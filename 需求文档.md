## 1. 项目概述

### 1.1 项目背景

A股市场超过5000只股票，短线交易者需要在海量标的中快速定位兼具 **资金活跃度**、**市场人气** 和 **技术面强势** 的个股。人工逐一筛选效率极低且容易遗漏关键标的。

短线助手 (shortais) 是一套自动化筛选工具，每日从全市场股票中，通过 "资金 × 人气" 交集筛选 + 线性回归动量评分，输出 **最具短线爆发力的TOP 10强势股**，并生成可视化网页报告，3分钟完成从数据获取到投资决策参考的全流程。

### 1.2 核心设计理念

| 维度 | 含义 | 衡量指标 |
|------|------|----------|
| **资金** | 大资金介入是短线拉升的前提 | 当日成交额排名 |
| **人气** | 市场关注度决定跟风意愿 | 当日个股热度排名 |
| **强势** | 趋势已确立且持续性强的标的 | 25日线性回归动量评分 |

三者缺一不可：有资金没人气 → 庄股风险；有人气没资金 → 散户自嗨；有趋势没资金人气 → 阴涨不爆。

### 1.3 技术栈

| 组件 | 技术选型 | 用途 |
|------|----------|------|
| 数据获取 | `pywencai` | 问财API，获取成交额排名&热度排名 |
| 历史行情 | `akshare` | 获取个股日线数据，计算动量 |
| 数据处理 | `pandas` / `numpy` | 数据清洗、标准化、交集计算 |
| 机器学习 | `scikit-learn (LinearRegression)` | 25日线性回归动量评分 |
| 可视化 | `plotly` | K线图、动量分析图（自包含HTML） |
| 输出格式 | `openpyxl` (Excel) + HTML | 数据存档 + 可视化报告 |

## 2. 功能需求

### 2.1 模块架构

```
get_data.py ──→ data/{date}/raw_data.xlsx
                        │
screener.py  ──→ data/{date}/selected_stocks.xlsx
                        │
visualizer.py ──→ data/{date}/report.html
```

### 2.2 模块一：数据获取 (get_data.py)

**功能描述**：从问财平台获取全市场成交额排名和热度排名数据。

**输入**：
- 自动取当日日期（可扩展指定历史日期）

**处理流程**：
1. 调用 `pywencai.get(query=f'{date}成交额前{VOLUME_TOP_N}')` 获取成交额排名
2. 调用 `pywencai.get(query=f'{date}个股热度前{HOT_TOP_N}')` 获取热度排名
3. 列名标准化处理（`股票简称` → `name`, `成交额[date]` → `volume_amount`, `个股热度[date]` → `hot_value` 等）

**输出**：
- `data/{YYYYMMDD}/raw_data.xlsx`
  - Sheet1 "成交额排名"：code, name, volume_amount, rank, pct_change, close 等
  - Sheet2 "热度排名"：code, name, hot_value, hot_rank, pct_change, close 等

**异常处理**：
- 网络请求间隔 ≥1s，避免API限流
- 数据为空时给出明确提示

---

### 2.3 模块二：强势股筛选 (screener.py)

**功能描述**：执行 "资金 × 人气" 交集筛选 + 动量评分，输出最终强势股名单。

**处理流程 — Phase A：交集 + 综合评分**

1. **找交集**：取成交额榜和热度榜的股票代码交集
2. **Min-Max 标准化**：
   - 对成交额列做标准化：$vol_{norm} = \frac{vol - vol_{min}}{vol_{max} - vol_{min}}$
   - 对热度值列做标准化：$hot_{norm} = \frac{hot - hot_{min}}{hot_{max} - hot_{min}}$
3. **综合评分**：$score_{composite} = \frac{vol_{norm} + hot_{norm}}{2}$
4. **排序取前 N**：按综合评分降序，取前 `FINAL_TOP_N`（默认30）只

> **为什么需要 Min-Max 标准化**：成交额单位为元（十亿级别），热度值为无量纲指数（百万级别），数值范围差异巨大，不标准化则成交额的权重会完全压倒热度，失去"双重筛选"意义。

**处理流程 — Phase B：动量评分**

1. 对 Phase A 选出的每只股票，通过 akshare 获取最近 `MOMENTUM_DAYS`（默认25）个交易日的收盘价
2. 计算相对价格序列：$p_{rel}[i] = \frac{close[i]}{close[0]}$（以第一天为基准）
3. 线性回归拟合：$y = slope \times x + intercept$，其中 $x = [0, 1, 2, ..., 24]$, $y = 相对价格$
4. **动量评分**：$momentum = 10000 \times slope \times R^2$

> **为什么用 $slope \times R^2$**：
> - **斜率 (slope)**：衡量趋势强度，斜率越大说明价格上涨越快
> - **决定系数 ($R^2$)**：衡量趋势可靠性，$R^2$ 越接近1说明走势越"规整"（沿趋势线稳定运行）
> - 两者相乘，确保只有 **既强劲又稳定** 的上涨趋势才能得高分
> - 纯价格涨幅会被单日暴涨误导，回归线则综合考虑了一段时间的整体走势质量

5. 按动量评分降序，取前 `MOMENTUM_TOP_N`（默认10）只

**输出**：
- `data/{YYYYMMDD}/selected_stocks.xlsx`
  - Sheet1 "全部候选"：30只候选股完整评分数据
  - Sheet2 "TOP{N}"：最终N只强势股（含综合评分、动量评分、趋势斜率、趋势R²）

**异常处理**：
- 交集为空：抛出明确错误提示
- 历史数据不足（新股/停牌）：该股标记为无数据，不计入最终排名
- API请求间隔 ≥0.15s，避免限流

### 2.4 模块三：可视化报告 (visualizer.py)

**功能描述**：读取筛选结果，生成包含K线图和动量分析的自包含HTML网页报告。

**输入**：`data/{YYYYMMDD}/selected_stocks.xlsx` 中的 TOP{N} sheet

**报告内容**：

| 区域 | 内容 |
|------|------|
| 页面头部 | 报告标题、日期、算法说明、精选数量 |
| 汇总表格 | 排名、名称、代码、综合评分、动量评分、趋势斜率、R² |
| 个股卡片 × N | 每只股票一张卡片，内含两个图表 |

**每只股票的图表**：
1. **动量分析图**（双面板）：
   - 上：相对价格走势 + 线性回归趋势线（标注斜率和R²）
   - 下：实际收盘价柱状图（红涨绿跌）
2. **K线图**（双面板）：
   - 上：60日K线（OHLC蜡烛图）+ MA5/MA10/MA20均线
   - 下：成交量柱状图（红涨绿跌）

**输出**：`data/{YYYYMMDD}/report.html` — 自包含HTML文件（Plotly CDN加载JS），浏览器直接打开即可

**异常处理**：
- K线数据获取失败：显示 "数据不足" 占位图，不阻塞报告生成
- API请求含重试机制（3次重试，间隔1秒）
- 请求间隔 ≥0.5s，降低限流概率

## 3. 配置参数

所有可调参数集中在 `config.py`：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `VOLUME_TOP_N` | 100 | 成交额排名取前N只 |
| `HOT_TOP_N` | 100 | 热度排名取前N只 |
| `FINAL_TOP_N` | 30 | 综合评分取前N名进入动量分析 |
| `MOMENTUM_DAYS` | 25 | 动量计算回看天数（≈5周） |
| `MOMENTUM_TOP_N` | 10 | 最终输出股票数量 |
| `KLINE_DAYS` | 60 | 可视化K线图回看天数（≈3个月） |
| `OUTPUT_DIR` | "data" | 数据输出根目录 |

## 4. 数据流与运行方式

### 4.1 运行步骤

```bash
# 1. 安装依赖（仅首次）
pip install -r requirements.txt

# 2. 获取当日数据（成交额TOP100 + 热度TOP100）
python get_data.py

# 3. 筛选强势股（交集→综合评分→动量评分→TOP10）
python screener.py

# 4. 生成可视化报告
python visualizer.py

# 5. 浏览器打开查看
start data\20260527\report.html
```

### 4.2 数据流图

## 6. 非功能需求

### 6.1 性能
- 数据获取 (get_data.py)：约10-20秒（2次API调用）
- 筛选 (screener.py)：约30-60秒（30次历史数据API调用）
- 可视化 (visualizer.py)：约30-90秒（10次K线数据API调用，含重试）

### 6.2 可靠性
- 数据获取和K线请求均含重试机制
- 单个股票数据失败不影响整体流程
- 模块间通过Excel文件解耦，可独立运行和调试

### 6.3 可维护性
- 所有参数集中在 `config.py`，修改后无需改动业务代码
- 模块职责单一：获取 / 筛选 / 可视化各司其职
- 输出文件按日期分目录存储，历史数据自然归档

### 6.4 兼容性
- Python 3.10+
- Windows / macOS / Linux 均可运行
- HTML报告无需服务器，浏览器直接打开

## 7. 风险与局限

| 风险 | 影响 | 应对 |
|------|------|------|
| pywencai API 失效 | 数据获取失败 | 预留 adata/baostock 等替代数据源接口 |
| akshare 源站限流 | K线数据部分缺失 | 重试机制 + 占位图 + 增加请求间隔 |
| 非交易日运行 | 无当日数据 | 提示用户检查日期 |
| 极少数股票无交集 | 筛选池过小 | 可调大 `VOLUME_TOP_N` 和 `HOT_TOP_N` |
